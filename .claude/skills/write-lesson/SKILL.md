# 编写课程：$ARGUMENTS

你是一位资深教育专家。现在请为 ReadBooksWithAI 项目编写一课自含式教材。

## 参数解析

将 `$ARGUMENTS` 拆分为三个部分：
- `BOOK_ID`：书名目录（如 `c-and-pointers`）
- `MODULE`：模块目录（如 `module-01-memory-model`）
- `LESSON`：课程文件名（如 `lesson-01-types-and-memory`）

目标文件路径：`books/$BOOK_ID/$MODULE/$LESSON.md`

---

## 前置检查

1. 检查 `books/$BOOK_ID/BOOK-STYLE.md` 是否存在
   - **若不存在**：提示用户先运行 `/config-book $BOOK_ID`，停止
2. 检查 `books/$BOOK_ID/BOOK.md` 是否存在
   - **若不存在**：提示用户先运行 `/plan-book $BOOK_ID`，停止
3. 检查 `books/$BOOK_ID/$MODULE/MODULE.md` 是否存在
   - **若不存在**：提示用户检查模块名是否正确，或运行 `/plan-book $BOOK_ID`，停止
4. 检查 `books/$BOOK_ID/module-index.md` 中是否有该 lesson 条目
   - **若不存在**：提示用户检查课程名，停止
5. 检查目标文件 `books/$BOOK_ID/$MODULE/$LESSON.md` 是否已存在
   - **若已存在**：展示现有文件概要（frontmatter + 概念列表），询问"重写"还是"取消"
   - 若重写：备份为 `$LESSON.backup.md`，继续

---

## 执行流程

严格按以下顺序执行，不可跳步。

### 第一步：读取上下文

按以下顺序读取，建立完整的写课上下文：

1. `guides/lesson-writing-guide.md` — 通用写作规范
2. `guides/depth-system.md` — 深度标记规范
3. `books/$BOOK_ID/BOOK.md` — 目标读者、深度设定（`target_depth`）、学科领域
4. `books/$BOOK_ID/BOOK-STYLE.md` — **本书写作约束**（代码规范、隐性知识清单、图表偏好、⭐ 触发条件、术语翻译）
5. `books/$BOOK_ID/module-index.md` — 本课在整体中的位置、前后课关系、本课的核心概念列表
6. `books/$BOOK_ID/$MODULE/MODULE.md` — 本模块目标、课程依赖关系
7. `guides/terminology-glossary.md` — 已有术语的标准译名
8. 如果本课有前置课程（非模块第一课或有跨模块依赖），读取前序课程确认知识衔接

### 第二步：精准定位原书内容 + 搜索外部资源

#### 2.1 通过索引定位原书内容

根据 `module-index.md` 中本课的核心概念和对应的原书章节：

1. 读取 `books/$BOOK_ID/refs/` 下的相关索引文件（`*-index.md`）
2. 在索引的「概念→章节反查表」中匹配本课概念，找到**推荐页**
3. **只读取推荐页**，不读整章。推荐页不够时再扩展到索引标注的完整页码范围
4. ⚠️ **防过载**：每次不超过 20 页 PDF
5. 索引中标注"无需再读 PDF 原文"的内容，直接引用索引

> 本地参考资料是权威源，优先级高于网络搜索。冲突时以本地资料为准。

#### 2.2 搜索外部资源

从 BOOK.md 读取学科领域，**动态生成搜索关键词**（不要硬编码任何特定学科的搜索词）：

1. 对每个核心概念：
   - 用 WebSearch 搜索 `[概念名] [学科领域] 详解`（中文资源）
   - 用 WebSearch 搜索 `[概念名] [学科英文] tutorial`（英文资源）
2. 用 WebFetch 验证 URL 可访问且内容相关
3. **降级策略**：WebFetch 遇到 403/反爬/超时，立即放弃，不反复重试。若外部搜索无高质量结果，依赖本地资料和自身知识库
4. 记录有效资源（名称、URL、一句话说明）

资源质量优先级：
1. 本地参考资料（`refs/` 索引定位的内容）
2. 官方文档
3. 知名教程站
4. 高质量技术博客
5. 大学公开课视频
6. Stack Overflow / 知乎等问答平台经典回答

#### 2.3 对比原书内容与外部资源，标记过时/可补充内容

将 2.1 读到的原书内容与 2.2 搜到的外部资源逐概念对比，判断：

1. **过时内容**：原书的工具、写法、API、最佳实践是否已被淘汰或有更好的替代？
   - 若过时：记录 `[概念] — 原书说法 → 现代替代 → 来源`
2. **原书解释不清的概念**：外部资源中是否有更好的类比、图解、解释角度？
   - 若有：记录 `[概念] — 可补充的更好解释 → 来源`
3. **原书遗漏**：该概念在当前实践中有重要的注意事项或常见用法，但原书未提及？
   - 若有：记录 `[概念] — 原书未提及的要点 → 来源`

> **处理原则**：
> - 原书内容**正确且未过时**的部分，以原书为权威，外部资源仅作补充
> - 原书内容**已过时**的部分，在正文中用对比方式说明（"原书写法 → 现代写法"），不是默默替换
> - 外部资源提供了**更好的解释方式**，可以吸收融入正文讲解中，注明灵感来源
> - 所有对比判断在第三步的大纲中体现，让用户在确认大纲时看到哪些地方做了更新

### 第三步：搭建课程骨架（先结构，后内容）

⚠️ **必须分两个阶段完成，禁止一步到位。**

#### 阶段 A：生成大纲骨架

根据 `module-index.md` 中本课的核心概念和 `guides/lesson-writing-guide.md` 的文档结构，输出一份**纯结构大纲**，包含：

1. **YAML frontmatter** 草案（module、lesson、title、original_chapters、prerequisites、concepts 等）
2. **前置知识检查**的具体问题（2-3 个，基于前序课程已教内容）
3. **核心概念列表**：每个概念列出标题 + 一句话定位（"这一节要解决什么问题"）
4. 每个概念下标注需要的子项：`[是什么] [为什么] [代码演示] [易错点] [⭐深入（如有）] [BEGINNER（如有）]`
5. **内容更新标注**（第 2.3 步的对比结果）：哪些概念的原书内容需要更新或补充，逐条列出：
   - 🔄 **过时更新**：`[概念] 原书用了 X，现代做法是 Y`
   - 📝 **解释增强**：`[概念] 融入来自 Z 的更好解释角度`
   - ➕ **原书遗漏**：`[概念] 补充原书未提及的 W`
   - ✅ **无需更新**的概念也标注，让用户一目了然
6. **图表计划**：标注在哪个概念下插入什么类型的图
7. **陷阱清单**的条目标题（至少 3 条）
8. **自测题**草案（2-3 道思考题）
9. **资源列表**（第二步收集的有效资源）

将大纲**写入 `books/$BOOK_ID/$MODULE/$LESSON-outline.md`**，然后提示用户查看并确认。

**等待用户确认后再进入阶段 B。**

#### 阶段 B：逐节填充内容

用户确认大纲后，按骨架**逐个概念**展开编写完整课程。

内容要素（从 `guides/lesson-writing-guide.md` 读取完整规范，以下为核心检查项）：

- **YAML frontmatter**：完整元数据（格式见 `CLAUDE.md`）
- **前置知识检查**：基于前序课程，引用具体课程文件的相对路径
- **核心概念**（每个概念遵循四段式）：
  - **是什么**：文字解释 + 图表（Mermaid 优先，ASCII 降级）
  - **为什么重要**：解决什么问题，实际中哪里用到
  - **代码演示**：按 `BOOK-STYLE.md` 一、代码规范的完整性要求和编译/运行命令
  - **内容更新**（若大纲中标注了该概念需要更新）：
    - 🔄 过时内容：在正文中用对比方式呈现（"原书写法 → 现代写法"），说明为什么变了
    - 📝 解释增强：将更好的解释融入"是什么"或"为什么重要"中，自然衔接
    - ➕ 原书遗漏：作为独立小节或融入相关段落，标注"原书未提及"
  - **易错点**：至少一组 ❌ 错误 + ✅ 正确对比
  - **⭐ 深入**（可选）：按 `BOOK-STYLE.md` 四、⭐ 触发条件判断是否标记，按 `BOOK.md` 的 `target_depth.advanced` 控制展开程度
  - **[BEGINNER]**（可选）：按 `BOOK-STYLE.md` 五、BEGINNER 场景判断是否添加，按 `BOOK.md` 的 `target_depth.beginner` 控制展开程度
- **概念串联**：本课概念之间的关系，与前后课的衔接
- **常见陷阱清单**：至少 3 条，Markdown 表格（# | 陷阱 | 症状 | 原因 | 修复）
- **动手练习提示**：给思路，不给完整答案
- **自测题**：2-3 道思考题，不给答案
- **补充资源**：第二步验证过的有效资源，表格格式

### 第四步：质量自查

在输出最终文件前，静默按以下清单逐项检查（有不符立即修正，不输出检查过程）：

- [ ] 每个概念集齐"是什么 + 为什么 + 代码/示例 + 易错点"
- [ ] 代码/示例符合 `BOOK-STYLE.md` 的完整性要求和编译/运行命令
- [ ] 每个概念至少一组正反对比（❌ + ✅）
- [ ] 大纲中标注的内容更新（🔄📝➕）已全部落地到正文中
- [ ] 至少一张图（类型参考 `BOOK-STYLE.md` 图表偏好）
- [ ] ⭐ 标记符合 `BOOK-STYLE.md` 触发条件，展开程度符合 target_depth
- [ ] [BEGINNER] 标记符合 `BOOK-STYLE.md` 补充场景，展开程度符合 target_depth
- [ ] `BOOK-STYLE.md` 隐性知识清单中标注本模块首次涉及的条目已覆盖
- [ ] 术语翻译符合 `BOOK-STYLE.md` 术语偏好
- [ ] 前置知识检查基于前序课程已教内容（不超纲）
- [ ] 陷阱清单 ≥ 3 条，表格格式
- [ ] 自测题 2-3 道
- [ ] 新术语首次出现时有解释，翻译一致
- [ ] 体量不少于 300 行，超过 1000 行时评估是否需要拆课
- [ ] 所有资源链接公开可访问或标注为书籍/搜索关键词

### 第五步：写入文件

1. 将完整课程保存到 `books/$BOOK_ID/$MODULE/$LESSON.md`
2. 删除大纲文件 `books/$BOOK_ID/$MODULE/$LESSON-outline.md`

### 第六步：更新全局状态

1. **术语表**：新术语追加到 `guides/terminology-glossary.md`
2. **课程索引**：将 `books/$BOOK_ID/module-index.md` 中对应课程的状态从 `⬜` 更新为 `📝`

---

## 体量与风格速查

- **体量**：内容完整性优先于行数。不少于 300 行，无硬上限。超过 1000 行时考虑拆课
- **语言**：主体中文，代码/命令/专业术语保留英文原文
- **人称**：用"你"称呼读者
- **深度**：以实用级为主，⭐ 深入和 [BEGINNER] 按 BOOK.md 的 target_depth 设定调节
