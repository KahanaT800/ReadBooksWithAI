# 深度标记系统

> 本文档定义 ReadBooksWithAI 课程中三档内容深度的标记规范。
> 三档内容共存于同一个 lesson.md 文件中，通过标记区分。
> 本规范适用于所有书籍，不绑定特定学科。

---

## 总览

| 档位 | 标记方式 | 目标读者 | 展开程度 |
|------|---------|---------|---------|
| 入门级 | `> [BEGINNER]` 块引用 | 零基础，需补充前置概念 | 简洁，1-3 句话 |
| 实用级 | 无标记（默认） | 有一定基础，能独立运用 | 完整展开 |
| 深入级 | `⭐ 深入：` 标题前缀 | 想了解底层原理 | 可选，可跳过 |

---

## 入门级 `[BEGINNER]`

### 用途

为零基础或跨领域读者补充"你可能不知道但需要知道"的前置概念。有经验的读者可以跳过。

### 格式

```markdown
> [BEGINNER] 如果你不知道什么是"变量"：变量就是程序中用来存储数据的命名容器，你给一块存储空间起个名字，之后就能通过名字读写里面的数据。
```

### 规则

- 用块引用（`>`）包裹
- 以 `[BEGINNER]` 开头
- 内容简洁，1-3 句话解决一个概念
- 不要在 BEGINNER 块中放代码示例（代码放在主体中）
- 每课的 BEGINNER 块不超过 3 个（太多说明课程拆分不合理）

---

## 实用级（默认）

### 用途

课程的主体内容。目标是让读者能独立运用所学知识。

### 格式

无特殊标记，就是正文。

### 深度标准

- 能说清楚"是什么、为什么、怎么用"
- 能识别和避免常见错误
- 能用自己的话向别人解释

---

## 深入级 `⭐ 深入`

### 用途

超出日常使用范畴的深层原理内容。跳过不影响后续学习。

### 格式

```markdown
#### ⭐ 深入：[底层原理标题]

> 以下内容为深层原理，理解它有助于加深认识，但不影响日常使用。跳过不影响后续学习。

（正文...）
```

### 规则

- 标题用 `⭐ 深入：` 前缀
- 紧跟一行灰色提示（块引用），告诉读者可以跳过
- 适用场景（根据学科灵活判断）：
  - 底层实现原理（如库/框架的内部机制）
  - 硬件/系统层面的行为
  - 工具链的具体实现细节
  - 标准规范的精确定义
  - 数学推导或理论证明
- 每课的 ⭐ 深入部分总计不超过课程体量的 20%
- 体量紧张时优先压缩此部分（保留标题和关键结论）

---

## BOOK.md 中的深度控制

每本书的 `BOOK.md` 文件中设置 `target_depth` 字段，控制 AI 写作时各档的展开程度：

```yaml
target_depth:
  beginner: moderate    # none / minimal / moderate
  standard: full        # full（始终完整展开）
  advanced: moderate    # none / minimal / moderate / full
```

| 值 | 含义 |
|---|------|
| `none` | 不生成该档内容 |
| `minimal` | 只保留标题和一句话总结 |
| `moderate` | 适度展开，保留关键信息 |
| `full` | 完整展开所有内容 |

AI 写课时读取此设定，据此调整各档篇幅。
