## 核心概念

### 1. 内存和地址

#### 是什么

计算机内存可以看成一个巨大的**字节数组**。每个字节（byte）包含 8 个位（bit），可以存储 0-255 的无符号值。每个字节都有一个唯一的编号，这个编号就是**地址**（address）。

```
        地址:    0x1000  0x1001  0x1002  0x1003  0x1004  0x1005  0x1006  0x1007
               +-------+-------+-------+-------+-------+-------+-------+-------+
    字节内容:  | 0x2A  | 0x00  | 0x00  | 0x00  | 0xFF  | 0xFF  | 0xFF  | 0xFF  |
               +-------+-------+-------+-------+-------+-------+-------+-------+
               |<-------- int a = 42 --------->|<-------- int b = -1 --------->|
               |       4 字节（32 位）          |       4 字节（32 位）          |
```

一个 `int` 通常占 4 个字节。虽然它跨越了 4 个地址，但我们只用**起始地址**来标识它。上图中 `a` 的地址是 `0x1000`，`b` 的地址是 `0x1004`。

> [BEGINNER] 十六进制 `0x` 表示法：地址通常以十六进制显示。`0x` 前缀表示后面的数字是十六进制。`0x1000` 就是十进制的 4096。每位十六进制对应 4 个二进制位，所以用来表示内存地址比十进制更紧凑。

高级语言的变量名是编译器提供的**别名**——你写 `a`，编译器翻译成"读取地址 `0x1000` 处的 4 字节"。硬件不认识变量名，只认地址。

关于内存，你只需要记住两件事：

1. 内存中的每个位置由一个**独一无二的地址**标识
2. 内存中的每个位置都**包含一个值**

#### 为什么重要

指针的本质就是存储地址的变量。如果不理解"地址"是什么，就无法理解指针。本节建立的内存模型会贯穿整个模块。

#### 代码演示

```c
/* memory_address.c — 查看变量的地址和大小 */
#include <stdio.h>

int main(void) {
    int a = 42;
    int b = -1;
    char c = 'X';
    double d = 3.14;

    /* %p 格式符打印指针（地址），& 取地址操作符 */
    printf("变量   地址              大小\n");
    printf("----   ----              ----\n");
    printf("a      %p    %zu 字节\n", (void *)&a, sizeof(a));
    printf("b      %p    %zu 字节\n", (void *)&b, sizeof(b));
    printf("c      %p    %zu 字节\n", (void *)&c, sizeof(c));
    printf("d      %p    %zu 字节\n", (void *)&d, sizeof(d));

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o memory_address memory_address.c
./memory_address
```

可能的输出（地址每次运行不同）：

```
变量   地址              大小
----   ----              ----
a      0x7ffd3a2b1c4c    4 字节
b      0x7ffd3a2b1c48    4 字节
c      0x7ffd3a2b1c47    1 字节
d      0x7ffd3a2b1c38    8 字节
```

注意观察：
- `int` 占 4 字节，`char` 占 1 字节，`double` 占 8 字节
- 相邻变量的地址之间的差距和 `sizeof` 大致对应（栈上变量可能有间隙，因为编译器会做对齐优化）
- 栈上变量的地址从高到低增长（这是 x86-64 的典型行为，但不是所有架构都如此）

#### 易错点

❌ **以为变量名就是地址：**

```c
int a = 42;
/* 错误想法："a 就是地址 0x1000" */
/* 实际：a 是编译器给的名字，编译后变量名消失 */
/* 你不能预测 a 会被放在哪个地址 */
```

✅ **正确理解：变量名是别名，`&` 才能取到地址：**

```c
int a = 42;
printf("a 的值: %d\n", a);       /* 42 — 变量名取值 */
printf("a 的地址: %p\n", (void *)&a); /* 0x... — & 取地址 */
```

#### ⭐ 深入：边界对齐

> 以下内容为深层原理，理解它有助于加深认识，但不影响日常使用。跳过不影响后续学习。

在要求**边界对齐**（boundary alignment）的机器上，多字节类型的起始地址必须是其大小的倍数。例如 4 字节的 `int` 通常存储在 4 的倍数地址上，8 字节的 `double` 存储在 8 的倍数地址上。这是硬件设计的要求——对齐访问比非对齐访问更快，有些架构甚至禁止非对齐访问（会触发总线错误 bus error）。

这解释了为什么上面示例中变量之间可能有"间隙"——编译器会插入填充字节来满足对齐要求。你在 module-05（结构体）中会详细学到内存对齐对 `sizeof` 的影响。

### 2. 值和类型

#### 是什么

内存中存储的只是一串 0 和 1——**位模式**（bit pattern）。同一个位模式，根据你告诉编译器的**类型**不同，会被解释为完全不同的值。

例如，4 字节的位模式 `01000000 01001000 11110101 11000011` 可以是：

| 解释为类型 | 值 |
|-----------|-----|
| `int` | 1078523331 |
| `float` | 3.14（近似） |
| 4 个 `char` | `@` `H` `õ` `Ã` |

类型信息**不存储在内存中**——它只存在于编译器的类型系统中。编译器根据你声明的类型，决定用什么指令（整型指令还是浮点指令）去读取那块内存。运行时，内存里只有 0 和 1，没有任何"标签"说明"这里存的是 int"。

#### 为什么重要

这个概念是理解**指针类型**的前提。指针变量存的也是一串位（地址值），指针的类型告诉编译器两件事：

1. 解引用时读多少字节（`int *` 读 4 字节，`char *` 读 1 字节）
2. 用什么指令解释这些字节（整型、浮点、字符……）

如果指针类型搞错了，读出来的值就是垃圾。

#### 代码演示

```c
/* value_and_type.c — 同一位模式的不同解释 */
#include <stdio.h>
#include <string.h>

int main(void) {
    float f = 3.14f;
    int i;

    /* 把 float 的位模式原样复制到 int 变量中 */
    memcpy(&i, &f, sizeof(i));

    printf("float 解释: %f\n", f);
    printf("int   解释: %d\n", i);
    printf("两者占用相同的 %zu 字节\n", sizeof(f));
    printf("但同一位模式被解释为完全不同的值！\n");

    /* 反过来：把一个整数的位模式当 float 看 */
    int magic = 1078523331;
    float result;
    memcpy(&result, &magic, sizeof(result));
    printf("\n整数 %d 的位模式按 float 解释: %f\n", magic, result);

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o value_and_type value_and_type.c
./value_and_type
```

输出：

```
float 解释: 3.140000
int   解释: 1078523331
两者占用相同的 4 字节
但同一位模式被解释为完全不同的值！

整数 1078523331 的位模式按 float 解释: 3.140000
```

#### 易错点

❌ **以为类型信息存储在内存中（像 Python/Java 那样）：**

```c
int a = 42;
/* 错误想法："内存中除了 42，还记录了 a 是 int 类型" */
/* 实际：内存中只有 42 对应的位模式 0x0000002A */
/* C 编译后没有任何运行时类型信息 */
```

✅ **正确理解：类型只在编译期有效，运行时只有位模式：**

```c
int a = 42;
float *fp = (float *)&a;  /* 强行用 float* 指向 int 的地址 */
printf("%f\n", *fp);       /* 得到垃圾值——同一位模式被错误解释 */
/* 编译器会警告，这就是类型系统在保护你 */
```

### 3. 指针声明与初始化

#### 是什么

**指针**（pointer）是一种变量，它存储的值是另一个变量的**地址**。

声明一个指针变量：

```c
int *p;    /* p 是一个指针，指向 int 类型 */
```

这里的 `*` 是**声明的一部分**，表示"p 是一个指针"。读法是："p 是一个指向 int 的指针"。

用 `&`（取地址操作符）获取一个变量的地址，赋给指针：

```c
int a = 42;
int *p = &a;   /* p 存储 a 的地址 */
```

此时内存中的关系：

```
变量 p                     变量 a
+------------+             +------+
| 0x7ffd1234 |------------>|  42  |
+------------+             +------+
  地址: 0x7ffd1240           地址: 0x7ffd1234
  大小: 8 字节               大小: 4 字节
  类型: int *                类型: int
```

关键事实：**指针变量本身也占内存**。在 64 位系统上，任何类型的指针都占 8 字节（因为 64 位地址空间需要 8 字节来表示一个地址）。在 32 位系统上是 4 字节。

```c
sizeof(int *)     == 8   /* 64 位系统 */
sizeof(char *)    == 8   /* 同上 */
sizeof(double *)  == 8   /* 同上 */
```

指针的类型（`int *`、`char *`、`double *`）不影响指针本身的大小，但决定了解引用时怎么解释目标地址的内容。

#### 为什么重要

指针让你能**间接访问**数据。这带来三个核心能力：

1. **函数间共享数据**——C 是传值调用，传指针才能让函数修改调用者的变量（module-02 详讲）
2. **动态数据结构**——链表、树等结构的节点通过指针连接（module-07 详讲）
3. **高效操作大对象**——传一个 8 字节的指针比复制一个几 KB 的结构体快得多

#### 代码演示

```c
/* pointer_basics.c — 指针声明、初始化和基本操作 */
#include <stdio.h>

int main(void) {
    int a = 42;
    int *p = &a;   /* p 指向 a */

    printf("=== 变量 a ===\n");
    printf("a 的值:   %d\n", a);
    printf("a 的地址: %p\n", (void *)&a);
    printf("a 的大小: %zu 字节\n\n", sizeof(a));

    printf("=== 指针 p ===\n");
    printf("p 的值（即 a 的地址）: %p\n", (void *)p);
    printf("p 自己的地址:          %p\n", (void *)&p);
    printf("p 的大小:              %zu 字节\n\n", sizeof(p));

    /* 验证 p 的值等于 &a */
    printf("p == &a ? %s\n", (p == &a) ? "是" : "否");

    /* 所有指针类型大小相同 */
    printf("\n=== 不同类型指针的大小（64 位系统）===\n");
    printf("sizeof(int *)    = %zu\n", sizeof(int *));
    printf("sizeof(char *)   = %zu\n", sizeof(char *));
    printf("sizeof(double *) = %zu\n", sizeof(double *));
    printf("sizeof(void *)   = %zu\n", sizeof(void *));

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o pointer_basics pointer_basics.c
./pointer_basics
```

输出（地址值因运行而异）：

```
=== 变量 a ===
a 的值:   42
a 的地址: 0x7ffd3a2b1c4c
a 的大小: 4 字节

=== 指针 p ===
p 的值（即 a 的地址）: 0x7ffd3a2b1c4c
p 自己的地址:          0x7ffd3a2b1c40
p 的大小:              8 字节

p == &a ? 是

=== 不同类型指针的大小（64 位系统）===
sizeof(int *)    = 8
sizeof(char *)   = 8
sizeof(double *) = 8
sizeof(void *)   = 8
```

#### 易错点

❌ **多变量声明时只有第一个是指针：**

```c
/* multi_decl_trap.c — 多变量声明陷阱 */
#include <stdio.h>

int main(void) {
    int *p1, p2;  /* ⚠️ p1 是 int*，但 p2 只是 int！ */

    printf("sizeof(p1) = %zu\n", sizeof(p1));  /* 8（指针）*/
    printf("sizeof(p2) = %zu\n", sizeof(p2));  /* 4（int）*/

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o multi_decl_trap multi_decl_trap.c
./multi_decl_trap
```

输出：

```
sizeof(p1) = 8
sizeof(p2) = 4
```

C 的声明语法中，`*` 是和变量名绑定的，不是和类型名绑定的。`int *p1, p2;` 中的 `*` 只修饰 `p1`。

✅ **正确：每个变量名前都加 `*`，或分行声明：**

```c
int *p1, *p2;       /* 方式一：每个都加 * */

int *p1;            /* 方式二：分行声明（推荐）*/
int *p2;
```

### 4. 间接访问（解引用）

#### 是什么

通过指针访问它所指向地址的值，叫做**间接访问**（indirection）或**解引用**（dereference）。使用 `*` 操作符：

```c
int a = 42;
int *p = &a;
printf("%d\n", *p);   /* 42 — 解引用 p，访问 a 的值 */
```

`*p` 的含义是："去 p 存储的那个地址，读取（或写入）那里的值"。

`&`（取地址）和 `*`（解引用）是**互逆操作**：

```
    &a  →  得到 a 的地址（指针）
    *p  →  得到 p 指向的值

    *(&a)  ==  a    （先取地址，再解引用 → 回到原值）
    &(*p)  ==  p    （先解引用，再取地址 → 回到原指针）
```

图示：

```
          &（取地址）
    a  ─────────────────>  &a（地址，即 p 的值）
    ↑                        |
    |      *（解引用）        |
    +────────────────────────+

    "& 和 * 互逆：走出去再走回来，回到原点"
```

**`*p` 是左值**——这一点非常重要。既然 `*p` 标识了一个内存位置（a 的位置），它就可以出现在赋值号左边：

```c
*p = 100;   /* 通过指针修改 a 的值！a 变成 100 */
```

这里 `*p` 出现在赋值号左边时是左值（表示 a 的内存位置），出现在右边时是右值（表示 a 的值）：

```c
*p = 10 - *p;
/* 右边的 *p：右值，取 a 的当前值 */
/* 左边的 *p：左值，指定写入 a 的位置 */
/* 结果：a = 10 - a */
```

#### 为什么重要

解引用是指针的**核心操作**。没有它，指针就只是一个存了数字的变量，毫无用处。有了它，你才能：

- 通过指针**读取**另一个变量的值
- 通过指针**修改**另一个变量的值
- 在函数中通过指针参数修改调用者的变量（module-02 的核心主题）

#### 代码演示

```c
/* dereference.c — 解引用操作的完整演示 */
#include <stdio.h>

int main(void) {
    int a = 42;
    int *p = &a;

    /* 1. 通过指针读取值 */
    printf("a  = %d\n", a);    /* 直接访问 */
    printf("*p = %d\n", *p);   /* 间接访问（解引用）*/
    printf("两者相同: %s\n\n", (a == *p) ? "是" : "否");

    /* 2. 通过指针修改值 */
    *p = 100;
    printf("执行 *p = 100 后:\n");
    printf("a  = %d\n", a);    /* a 也变成 100 了！ */
    printf("*p = %d\n\n", *p);

    /* 3. & 和 * 互逆 */
    printf("*(&a) = %d（先取地址，再解引用 = a）\n", *(&a));
    printf("&(*p) = %p（先解引用，再取地址 = p）\n", (void *)&(*p));
    printf("p     = %p\n\n", (void *)p);

    /* 4. *p 作为左值和右值 */
    *p = 10 - *p;  /* 右边 *p 是右值(100)，左边 *p 是左值 */
    printf("执行 *p = 10 - *p 后:\n");
    printf("a = %d\n", a);  /* -90 */

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o dereference dereference.c
./dereference
```

输出：

```
a  = 42
*p = 42
两者相同: 是

执行 *p = 100 后:
a  = 100
*p = 100

*(&a) = 100（先取地址，再解引用 = a）
&(*p) = 0x7ffd3a2b1c4c（先解引用，再取地址 = p）
p     = 0x7ffd3a2b1c4c

执行 *p = 10 - *p 后:
a = -90
```

#### 易错点

❌ **混淆声明中的 `*` 和表达式中的 `*`：**

```c
int *p = &a;   /* 声明：* 是类型的一部分，"p 是 int 指针" */
*p = 100;      /* 表达式：* 是解引用操作符，"访问 p 指向的值" */
```

这两个 `*` 长得一样，但意思完全不同！

初学者容易犯的错误：

```c
int *p;
*p = &a;     /* ❌ 错误！*p 是 int 类型，&a 是 int* 类型 */
p = &a;      /* ✅ 正确！p 是 int* 类型，&a 也是 int* 类型 */
```

✅ **记住区分规则：在声明语句中 `*` 表示"这是指针类型"，在其他表达式中 `*` 表示"解引用"。**

❌ **以为指针变量的值就是它指向的数据：**

```c
int a = 42;
int *p = &a;
printf("%d\n", p);    /* ❌ p 的值是地址（如 0x7ffd1234），不是 42 */
printf("%d\n", *p);   /* ✅ *p 才是 42 */
```

✅ **牢记：指针变量和其他变量一样，值就是存在它自己内存位置里的那个数。对指针来说，那个数是一个地址。只有加上 `*` 操作符，才会"跟着地址走过去"读取目标值。**

#### ⭐ 深入：指针常量与硬件编程

> 以下内容为深层原理，理解它有助于加深认识，但不影响日常使用。跳过不影响后续学习。

`*` 操作符只能作用于**指针类型**的表达式。整数字面量不是指针类型：

```c
*100 = 25;   /* ❌ 编译错误：100 是 int，不是指针 */
```

如果你确实需要访问某个特定地址（极罕见），必须用强制类型转换：

```c
*(int *)100 = 25;  /* 把 100 从 int 转为 int*，再解引用 */
```

这种用法几乎只出现在**嵌入式/操作系统编程**中——通过特定地址与硬件设备通信（如内存映射 I/O）。日常应用开发中永远不会用到。

为什么不能直接用整数地址？因为你无法预测编译器会把变量放在哪个地址。变量的地址只能通过 `&` 操作符在运行时获取。

- **内存和地址**告诉你：内存是字节数组，每字节有地址
- **值和类型**告诉你：类型决定如何解读内存中的位模式——这解释了为什么指针需要类型
- **指针声明**告诉你：可以用变量来存储地址，`&` 获取地址
- **间接访问**告诉你：用 `*` 通过地址访问目标数据

## 常见陷阱清单

| # | 陷阱 | 症状 | 原因 | 修复 |
|---|------|------|------|------|
| 1 | `int *p1, p2;` 多变量声明 | `p2` 不能像指针一样使用，编译报类型错误 | `*` 只修饰紧跟的变量名，`p2` 是 `int` 不是 `int *` | 每个变量名前加 `*`：`int *p1, *p2;` 或分行声明 |
| 2 | 混淆声明 `*` 和解引用 `*` | 写出 `*p = &a;` 导致类型不匹配 | 声明中 `*` 表示指针类型，表达式中 `*` 是解引用操作符 | 赋地址给指针用 `p = &a;`（不加 `*`），通过指针写值用 `*p = 100;` |
| 3 | 打印指针用 `%d` | 输出截断或乱码（64 位地址被截断为 32 位 int） | `%d` 期望 int，指针是 8 字节地址 | 用 `%p` 格式符并强转 `(void *)` |
| 4 | 以为指针值就是目标数据 | 调试时看到指针的值是个大数而困惑 | 指针的值是地址（一个数字），不是它指向的数据 | 打印 `p` 看地址，打印 `*p` 看数据，区分清楚 |
| 5 | 忘记用 `&` 取地址 | 编译器报"incompatible types"或"makes pointer from integer" | 把变量值当地址赋给指针 | 赋值时写 `p = &a;` 不要写 `p = a;` |

## 动手练习提示

### 练习 1：内存侦探

写一个程序，声明 `int`、`short`、`long`、`char`、`double` 各一个变量，打印每个变量的地址和大小。观察：
- 不同类型的大小分别是多少？
- 变量在内存中是连续排列的吗？地址之间有间隙吗？
- 思路提示：用 `&` 取地址，`sizeof` 取大小，`%p` 打印地址
- 容易卡住的地方：`printf` 的 `sizeof` 用 `%zu` 格式符

### 练习 2：指针三连问

声明 `int a = 10; int *p = &a; int **pp = &p;`，然后打印：
- `a`、`*p`、`**pp` 的值
- `&a`、`p`、`*pp` 的值
- `&p`、`pp` 的值
- 验证每一层的关系：`pp` 指向 `p`，`p` 指向 `a`
- 思路提示：画出三个变量的内存图，标注地址和箭头
- 容易卡住的地方：`**pp` 是两层解引用——先解引用 `pp` 得到 `p`，再解引用 `p` 得到 `a`

### 练习 3：swap 函数

写一个函数 `void swap(int *x, int *y)` 交换两个整数的值。在 `main` 中调用它并验证交换成功。
- 思路提示：需要一个临时变量保存其中一个值
- 容易卡住的地方：函数参数是指针，函数内部要用 `*x` 和 `*y` 操作目标值
