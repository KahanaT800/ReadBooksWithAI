## 核心概念

### 1. 未初始化指针

#### 是什么

声明一个指针变量但不初始化，它里面存的是**垃圾值**——一个随机的地址。对这样的指针执行解引用，就是让程序去访问一个你完全不知道的内存位置。这是**未定义行为**（UB）。

```
  未初始化指针                        正确初始化的指针

  +--------+                          +--------+
  |  ???   |----> ??? （随机位置）     |  &a    |----> a = 42
  +--------+      可能是:              +--------+
  int *p;         - 操作系统保留区域   int *p = &a;
                  - 其他变量的位置
                  - 根本不存在的地址
```

注意：**指针变量和其他变量遵循同样的初始化规则**——如果是静态或全局变量，自动初始化为 0（即 NULL）；如果是局部（自动）变量，**不会被初始化**，包含垃圾值。

#### 为什么重要

未初始化指针是 C 程序中**最危险的 bug 之一**，因为它可能导致两种后果：

1. **好的情况**：垃圾地址恰好是非法地址，程序立即崩溃（段错误），你能定位问题
2. **坏的情况**：垃圾地址恰好指向某个合法位置，程序**悄悄修改了别的数据**，但不崩溃——这种 bug 可能在完全不相关的代码中才暴露出来，极难排查

原书的话说得很直白："如果你运气好，会立刻段错误；运气不好，你会无声无息地破坏其他数据。"

#### 代码演示

```c
/* uninit_ptr.c — 未初始化指针的危险 */
#include <stdio.h>

int main(void) {
    int *p;  /* 未初始化！p 含垃圾地址 */

    /* ❌ 解引用未初始化的指针 — 未定义行为 */
    /* 取消下面这行的注释会导致段错误（或更糟：静默破坏数据）*/
    /* *p = 42; */

    printf("未初始化的 p 的值: %p（垃圾地址）\n", (void *)p);
    printf("这个地址指向哪里？不知道！解引用它是 UB。\n");

    /* ✅ 正确做法：先初始化再使用 */
    int a = 100;
    p = &a;      /* 现在 p 指向一个有效的位置 */
    printf("\n初始化后 p = %p，*p = %d\n", (void *)p, *p);

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o uninit_ptr uninit_ptr.c
./uninit_ptr
```

输出（`p` 的垃圾值每次不同）：

```
未初始化的 p 的值: 0x7ffd3a2b1c40（垃圾地址）
这个地址指向哪里？不知道！解引用它是 UB。

初始化后 p = 0x7ffd3a2b1c3c，*p = 100
```

> 编译器可能会对未初始化变量发出警告（`-Wuninitialized`）。**永远不要忽略这些警告。**

#### 易错点

❌ **声明指针后直接写值——指针没有指向任何有效位置：**

```c
int *p;
*p = 42;  /* ❌ UB！p 含垃圾地址 */
```

关键错误：声明 `int *p;` 只是分配了存放指针本身的内存（8 字节），**并没有分配存放 int 值的内存**。`*p = 42` 试图把 42 写入一个随机位置。

✅ **必须先让指针指向一个有效位置：**

```c
int a;
int *p = &a;  /* 方式一：指向已有变量 */
*p = 42;      /* ✅ 现在 a = 42 */

/* 或者用 malloc 分配堆内存（module-06 详讲） */
```

### 2. NULL 指针

#### 是什么

**NULL 指针**是一种特殊的指针值，表示"当前不指向任何有效数据"。要把指针设为 NULL，给它赋零值：

```c
int *p = NULL;   /* 或者 int *p = 0; */
```

`NULL` 是一个宏，通常定义为 `((void *)0)`。它的含义是明确的：**这个指针当前是无效的，不能解引用。**

对 NULL 指针解引用是未定义行为。在大多数现代系统上，这会导致**段错误**（segmentation fault），程序立即终止。这实际上是好事——比起悄悄破坏数据，立刻崩溃让你更容易定位 bug。

#### 为什么重要

NULL 指针解决了一个实际问题：**如何区分"指针有效"和"指针无效"？**

- 未初始化指针含垃圾值——你无法判断它是否有效
- NULL 指针有明确的值 0——你可以用 `if (p != NULL)` 检查

这在两个场景中特别有用：

1. **函数返回值**：函数找不到结果时返回 NULL，调用者检查返回值
2. **尚未使用的指针**：声明时赋 NULL，使用前检查

#### 代码演示

```c
/* null_ptr.c — NULL 指针的使用和检查 */
#include <stdio.h>
#include <stdlib.h>

/* 在数组中查找值，找到返回指向该元素的指针，找不到返回 NULL */
int *find_value(int arr[], int size, int target) {
    for (int i = 0; i < size; i++) {
        if (arr[i] == target) {
            return &arr[i];  /* 找到了，返回指向该元素的指针 */
        }
    }
    return NULL;  /* 没找到，返回 NULL */
}

int main(void) {
    int nums[] = {10, 20, 30, 40, 50};
    int size = sizeof(nums) / sizeof(nums[0]);

    /* 查找存在的值 */
    int *result = find_value(nums, size, 30);
    if (result != NULL) {
        printf("找到了: %d（地址 %p）\n", *result, (void *)result);
    } else {
        printf("没找到\n");
    }

    /* 查找不存在的值 */
    result = find_value(nums, size, 99);
    if (result != NULL) {
        printf("找到了: %d\n", *result);
    } else {
        printf("99 没找到，result = %p（NULL）\n", (void *)result);
    }

    /* ❌ 不检查就解引用——如果 result 是 NULL 会段错误 */
    /* printf("%d\n", *result);  — 段错误！ */

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o null_ptr null_ptr.c
./null_ptr
```

输出：

```
找到了: 30（地址 0x7ffd3a2b1c34）
99 没找到，result = (nil)（NULL）
```

**防御性编程最佳实践**（原书未强调但非常重要）：

```c
/* ✅ 声明指针时如果不知道指向谁，先赋 NULL */
int *p = NULL;

/* ✅ 使用前检查 */
if (p != NULL) {
    printf("%d\n", *p);
}

/* ✅ 简写形式（C 中 NULL == 0，在 if 中为假） */
if (p) {           /* 等价于 if (p != NULL) */
    printf("%d\n", *p);
}
```

#### 易错点

❌ **不检查就解引用可能为 NULL 的指针：**

```c
int *p = find_something();
printf("%d\n", *p);  /* ❌ 如果函数返回 NULL，这里段错误 */
```

✅ **解引用前必须检查：**

```c
int *p = find_something();
if (p != NULL) {
    printf("%d\n", *p);  /* ✅ 确认非 NULL 再解引用 */
} else {
    printf("未找到\n");
}
```

### 3. 指针与左值

#### 是什么

lesson-01 已经介绍了 `*p` 是左值。本节系统分析各种指针表达式的左值/右值性质。

**核心规则**：间接访问操作符 `*` 的结果是**左值**——因为它标识了一个具体的内存位置。而 `&` 操作符的结果是**右值**——因为它产生的地址值没有"住在"某个固定位置。

| 表达式 | 左值？ | 原因 |
|--------|--------|------|
| `a` | 是 | 变量，标识一个内存位置 |
| `&a` | **否** | `&` 的结果是地址值，没有固定存储位置 |
| `p` | 是 | 指针变量本身，标识一个内存位置 |
| `&p` | **否** | 同 `&a`，地址值没有固定存储位置 |
| `*p` | 是 | 间接访问标识了 p 指向的内存位置 |
| `*p + 1` | **否** | `+` 的结果是计算值，没有固定存储位置 |
| `*(p + 1)` | 是 | 间接访问标识了 p+1 指向的内存位置 |

判断一个表达式是否是左值，问自己："它标识了一个**明确的内存位置**吗？" 如果是，就是左值；如果只是一个临时计算结果，就不是。

#### 为什么重要

理解哪些表达式是左值，才知道**哪些能被赋值、哪些能取地址**。写复杂指针代码时，这决定了表达式能不能出现在赋值号左边。

```c
*p = 100;       /* ✅ *p 是左值 */
*(p + 1) = 200; /* ✅ *(p+1) 是左值 */
p + 1 = &a;     /* ❌ p+1 不是左值，编译错误 */
&a = p;         /* ❌ &a 不是左值，编译错误 */
```

#### 代码演示

```c
/* lvalue_ptr.c — 指针表达式的左值性质 */
#include <stdio.h>

int main(void) {
    int arr[] = {10, 20, 30};
    int *p = arr;  /* p 指向 arr[0] */

    /* 1. *p 是左值：可以赋值 */
    *p = 100;
    printf("arr[0] = %d（通过 *p 修改）\n", arr[0]);

    /* 2. *(p+1) 是左值：间接访问产生左值 */
    *(p + 1) = 200;
    printf("arr[1] = %d（通过 *(p+1) 修改）\n", arr[1]);

    /* 3. *p + 1 不是左值：只是一个计算结果 */
    int val = *p + 1;  /* 读取 *p 的值再加 1 → 101 */
    printf("*p + 1 = %d（计算结果，不是左值）\n", val);

    /* 对比两个容易混淆的表达式 */
    printf("\n关键区分：\n");
    printf("*p + 1  = %d（先解引用，再加 1 → 值）\n", *p + 1);
    printf("*(p+1)  = %d（先移动指针，再解引用 → 左值）\n", *(p + 1));

    /* 4. p 本身也是左值（变量），可以修改 */
    p = &arr[2];
    printf("\np 改指向 arr[2] 后: *p = %d\n", *p);

    /* 5. 以下是编译错误（取消注释可验证）*/
    /* &arr[0] = p;  — ❌ &arr[0] 不是左值 */
    /* p + 1 = &arr[2]; — ❌ p+1 不是左值 */

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o lvalue_ptr lvalue_ptr.c
./lvalue_ptr
```

输出：

```
arr[0] = 100（通过 *p 修改）
arr[1] = 200（通过 *(p+1) 修改）
*p + 1 = 101（计算结果，不是左值）

关键区分：
*p + 1  = 101（先解引用，再加 1 → 值）
*(p+1)  = 200（先移动指针，再解引用 → 左值）

p 改指向 arr[2] 后: *p = 30
```

#### 易错点

❌ **混淆 `*p + 1` 和 `*(p + 1)`：**

```c
int arr[] = {10, 20};
int *p = arr;

int x = *p + 1;   /* x = 11（arr[0] 的值加 1）*/
int y = *(p + 1);  /* y = 20（arr[1] 的值）*/
/* 两个表达式的结果完全不同！ */
```

✅ **记忆规则：`*` 的优先级高于 `+`，想先做加法要加括号。`*p + 1` 是"解引用的结果加 1"，`*(p+1)` 是"指针移动后再解引用"。**

### 4. 指针的指针

#### 是什么

指针变量和普通变量一样，占据内存中的一个位置，有自己的地址。所以可以用另一个指针指向它——这就是**二级指针**（pointer to pointer），也叫**指针的指针**。

```c
int a = 12;
int *p = &a;     /* p 指向 a */
int **pp = &p;   /* pp 指向 p */
```

内存关系：

```
变量 pp                   变量 p                   变量 a
+------------+            +------------+            +------+
| &p         |----------->| &a         |----------->|  12  |
+------------+            +------------+            +------+
  地址: 0x30                地址: 0x20                地址: 0x10
  类型: int **              类型: int *               类型: int

  pp 的值 = 0x20           p 的值 = 0x10             a 的值 = 12
  *pp = p = 0x10           *p = a = 12
  **pp = *p = a = 12
```

声明 `int **pp;` 的读法：`**pp` 的类型是 `int`，所以 `pp` 是"指向指向 int 的指针的指针"。

逐层解引用：
- `pp` → pp 的值（p 的地址）
- `*pp` → p 的值（a 的地址）
- `**pp` → a 的值（12）

#### 为什么重要

二级指针有两个核心使用场景：

1. **在函数中修改指针本身**——C 是传值调用，传 `int *p` 只能修改 `*p`（a 的值），不能修改 `p` 本身。传 `int **pp` 才能修改 `*pp`（即 p 的值）。这在 module-02（函数）和 module-07（链表插入）中会频繁使用。

2. **指针数组**——`char **argv` 就是指向 `char *` 数组的指针，命令行参数就用这种形式（module-08 详讲）。

#### 代码演示

```c
/* double_ptr.c — 二级指针演示 */
#include <stdio.h>

/* 用二级指针在函数中修改指针本身 */
void reset_ptr(int **pp, int *new_target) {
    *pp = new_target;  /* 修改 *pp，即修改调用者的指针变量 */
}

int main(void) {
    int a = 10, b = 20;
    int *p = &a;
    int **pp = &p;

    /* 逐层解引用 */
    printf("=== 逐层解引用 ===\n");
    printf("pp   = %p（p 的地址）\n", (void *)pp);
    printf("*pp  = %p（a 的地址，即 p 的值）\n", (void *)*pp);
    printf("**pp = %d（a 的值）\n\n", **pp);

    /* 验证等价关系 */
    printf("=== 等价关系验证 ===\n");
    printf("*pp == p  ? %s\n", (*pp == p) ? "是" : "否");
    printf("**pp == *p ? %s\n", (**pp == *p) ? "是" : "否");
    printf("**pp == a  ? %s\n\n", (**pp == a) ? "是" : "否");

    /* 通过二级指针修改 a 的值 */
    **pp = 99;
    printf("执行 **pp = 99 后: a = %d\n\n", a);

    /* 用函数修改指针本身 */
    printf("=== 函数中修改指针 ===\n");
    printf("修改前: p 指向 a, *p = %d\n", *p);
    reset_ptr(&p, &b);  /* 传 &p，让函数能修改 p */
    printf("修改后: p 指向 b, *p = %d\n", *p);

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o double_ptr double_ptr.c
./double_ptr
```

输出：

```
=== 逐层解引用 ===
pp   = 0x7ffd3a2b1c38（p 的地址）
*pp  = 0x7ffd3a2b1c4c（a 的地址，即 p 的值）
**pp = 10（a 的值）

=== 等价关系验证 ===
*pp == p  ? 是
**pp == *p ? 是
**pp == a  ? 是

执行 **pp = 99 后: a = 99

=== 函数中修改指针 ===
修改前: p 指向 a, *p = 99
修改后: p 指向 b, *p = 20
```

#### 易错点

❌ **解引用层数搞错：**

```c
int a = 10;
int *p = &a;
int **pp = &p;

printf("%d\n", *pp);   /* ❌ *pp 是 int*（地址），不是 int */
printf("%d\n", **pp);  /* ✅ **pp 才是 int 值 */
```

✅ **画图追踪每一层。每多一个 `*`，就沿箭头走一步：**

```
pp --(*pp)--> p --(*p)--> a
       ↑                   ↑
   一次解引用          二次解引用
   得到 p 的值         得到 a 的值
   (类型: int*)        (类型: int)
```

### 5. 指针表达式

#### 是什么

`*`（解引用）和 `++`/`--`（递增递减）经常组合使用，它们因**优先级和结合性**不同而产生不同行为。这是原书 6.11 节的核心内容——搞懂这几个组合，你就能读懂绝大多数指针代码。

关键的优先级规则：
- **后缀 `++`/`--` 的优先级高于前缀 `*`**（后缀在第 1 档，前缀 `*` 在第 2 档）
- **前缀 `*` 和前缀 `++` 优先级相同**，结合性从**右向左**

四个核心组合：

| 表达式 | 等价于 | 效果 | 谁被修改 |
|--------|--------|------|---------|
| `*cp++` | `*(cp++)` | 先取 `*cp`，后 `cp` 向后移动 | **指针** cp |
| `*++cp` | `*(++cp)` | 先 `cp` 向后移动，后取 `*cp` | **指针** cp |
| `++*cp` | `++(*cp)` | 取 `*cp` 的值加 1 | **指向的值** *cp |
| `(*cp)++` | — | 取 `*cp` 的值（返回旧值），后将 `*cp` 加 1 | **指向的值** *cp |

**最常用的是 `*cp++`**——它是 C 的惯用法，用于遍历数组或字符串：

```c
while (*p != '\0') {
    putchar(*p++);   /* 输出当前字符，指针移到下一个 */
}
```

#### 为什么重要

这些组合在 C 代码中**极其常见**，尤其是处理字符串和数组时。如果搞不清"移动的是指针还是值"，就会写出逻辑错误的代码。原书专门用了整整一节（6.11）来分析这些表达式，就是因为它们对初学者来说是巨大的困惑源。

#### 代码演示

```c
/* ptr_expr.c — 指针表达式的四种组合 */
#include <stdio.h>

int main(void) {
    /* 每种组合用独立的数组和指针，避免相互干扰 */

    /* === 1. *cp++ — 最常用！取值后移动指针 === */
    printf("=== *cp++ ===\n");
    {
        char arr[] = "ABCDE";
        char *cp = arr;

        /* *cp++ 等价于 *(cp++)：
           步骤 1: 保存 cp 的当前值（副本）
           步骤 2: cp 自增（指向下一个元素）
           步骤 3: 对副本执行 *（取原位置的值）*/
        char ch = *cp++;
        printf("ch = '%c'（取的是原位置 arr[0]）\n", ch);
        printf("cp 现在指向 arr[1] = '%c'\n\n", *cp);
    }

    /* === 2. *++cp — 先移动指针再取值 === */
    printf("=== *++cp ===\n");
    {
        char arr[] = "ABCDE";
        char *cp = arr;

        /* *++cp 等价于 *(++cp)：
           步骤 1: cp 自增（指向下一个元素）
           步骤 2: 对增值后的 cp 执行 *（取新位置的值）*/
        char ch = *++cp;
        printf("ch = '%c'（跳过 arr[0]，取 arr[1]）\n", ch);
        printf("cp 现在指向 arr[1] = '%c'\n\n", *cp);
    }

    /* === 3. ++*cp — 不移动指针，修改指向的值 === */
    printf("=== ++*cp ===\n");
    {
        char arr[] = "ABCDE";
        char *cp = arr;

        /* ++*cp 等价于 ++(*cp)：
           步骤 1: *cp 取得 arr[0] 的值 'A'
           步骤 2: 对这个值执行前缀 ++（'A'+1 = 'B'）
           步骤 3: 修改后的值写回 arr[0]
           指针 cp 不动！ */
        char ch = ++*cp;
        printf("ch = '%c'（'A' + 1 = 'B'）\n", ch);
        printf("arr[0] 被修改为 '%c'\n", arr[0]);
        printf("cp 仍然指向 arr[0]（指针没动）\n\n");
    }

    /* === 4. (*cp)++ — 取旧值，然后修改指向的值 === */
    printf("=== (*cp)++ ===\n");
    {
        char arr[] = "ABCDE";
        char *cp = arr;

        /* (*cp)++ ：
           步骤 1: *cp 取得 arr[0] 的值 'A'
           步骤 2: 返回旧值 'A' 作为表达式结果
           步骤 3: arr[0] 的值加 1（变成 'B'）
           指针 cp 不动！ */
        char ch = (*cp)++;
        printf("ch = '%c'（返回旧值 'A'）\n", ch);
        printf("arr[0] 被修改为 '%c'（后缀++生效）\n", arr[0]);
        printf("cp 仍然指向 arr[0]（指针没动）\n\n");
    }

    /* === 对比总结 === */
    printf("=== 总结 ===\n");
    printf("*cp++   → 移动指针，取旧位置的值（最常用）\n");
    printf("*++cp   → 移动指针，取新位置的值\n");
    printf("++*cp   → 不动指针，修改值，返回新值\n");
    printf("(*cp)++ → 不动指针，修改值，返回旧值\n");

    return 0;
}
```

```bash
gcc -std=c99 -Wall -Wextra -g -o ptr_expr ptr_expr.c
./ptr_expr
```

输出：

```
=== *cp++ ===
ch = 'A'（取的是原位置 arr[0]）
cp 现在指向 arr[1] = 'B'

=== *++cp ===
ch = 'B'（跳过 arr[0]，取 arr[1]）
cp 现在指向 arr[1] = 'B'

=== ++*cp ===
ch = 'B'（'A' + 1 = 'B'）
arr[0] 被修改为 'B'
cp 仍然指向 arr[0]（指针没动）

=== (*cp)++ ===
ch = 'A'（返回旧值 'A'）
arr[0] 被修改为 'B'（后缀++生效）
cp 仍然指向 arr[0]（指针没动）

=== 总结 ===
*cp++   → 移动指针，取旧位置的值（最常用）
*++cp   → 移动指针，取新位置的值
++*cp   → 不动指针，修改值，返回新值
(*cp)++ → 不动指针，修改值，返回旧值
```

**实际应用：`*p++` 遍历字符串**

```c
/* str_traverse.c — *p++ 的惯用法 */
#include <stdio.h>

/* 用 *p++ 惯用法实现字符串长度计算 */
int my_strlen(const char *s) {
    int len = 0;
    while (*s++ != '\0') {  /* 取当前字符判断，指针移到下一个 */
        len++;
    }
    return len;
}

int main(void) {
    const char *msg = "Hello";
    printf("长度: %d\n", my_strlen(msg));  /* 5 */
    return 0;
}
```

#### 易错点

❌ **混淆"移动指针"和"修改值"：**

```c
char arr[] = "ABC";
char *cp = arr;

/* 你想把 arr[0] 从 'A' 变成 'B'，写了： */
*cp++;     /* ❌ 这移动了指针！arr[0] 没变 */

/* 应该写： */
++*cp;     /* ✅ 或 (*cp)++ 或 *cp += 1 */
```

✅ **分析技巧：看 `++` 和 `*` 谁先作用于 `cp`。**

- `*cp++`：后缀 `++` 先绑定 `cp`，但返回旧值 → `*` 作用于旧 `cp` → **指针移动**
- `++*cp`：`*` 先作用于 `cp`（从右到左结合）→ `++` 作用于 `*cp` 的结果 → **值被修改**

拿不准时，加括号让意图明确：`*(cp++)` 或 `++(*cp)`。

## 常见陷阱清单

| # | 陷阱 | 症状 | 原因 | 修复 |
|---|------|------|------|------|
| 1 | 声明指针后不初始化就解引用 | 段错误或静默破坏数据 | 指针含垃圾地址，指向随机位置 | 声明时用 `&` 或 `NULL` 初始化 |
| 2 | 不检查 NULL 就解引用 | 段错误（SIGSEGV） | 函数可能返回 NULL，调用者未检查 | 解引用前 `if (p != NULL)` |
| 3 | 混淆 `*p++` 和 `++*p` | 期望修改值但移动了指针，或反之 | `*p++` 移动指针，`++*p` 修改值 | 加括号明确意图：`*(p++)` 或 `++(*p)` |
| 4 | 二级指针解引用层数错误 | 类型不匹配警告，值是地址而非数据 | `*pp` 是 `int*`，`**pp` 才是 `int` | 画图追踪每层指向，数清 `*` 的个数 |
| 5 | 对 `&a` 赋值 | 编译错误：lvalue required | `&a` 是右值（地址计算结果），没有固定存储位置 | `&a` 只能用于取地址，不能当左值 |

## 动手练习提示

### 练习 1：安全的除法函数

写一个函数 `int safe_div(int a, int b, int *result)`：如果 `b != 0`，把 `a/b` 写入 `*result` 并返回 1；如果 `b == 0`，返回 0 且不修改 `*result`。在 `main` 中调用并处理两种情况。
- 思路提示：函数内检查 `b` 和 `result` 是否有效
- 容易卡住的地方：别忘了检查 `result` 是否为 NULL

### 练习 2：字符串复制

用 `*dst++ = *src++` 的惯用法实现一个简化版的 `strcpy`。
- 思路提示：循环条件是 `*src != '\0'`，别忘了复制最后的 `'\0'`
- 容易卡住的地方：目标数组必须足够大

### 练习 3：通过二级指针交换两个指针

写一个函数 `void swap_ptrs(int **pp1, int **pp2)` 交换两个指针的值（不是它们指向的值）。在 `main` 中验证。
- 思路提示：和普通 swap 一样，只是操作的是 `*pp1` 和 `*pp2`
- 容易卡住的地方：注意传参时用 `&p1` 和 `&p2`
