> 本文件是 `/write-lesson` 编写课程时的核心参照。桥接通用写作规范（`guides/lesson-writing-guide.md`）和本书的具体需求。

## 一、代码规范

### 完整性要求

每个代码示例必须是**完整的、可直接编译运行的 C 程序**，包含：

1. 所有必要的 `#include` 头文件
2. `int main(void)` 入口函数（或 `int main(int argc, char *argv[])` 当涉及命令行参数时）
3. 合适的 `return 0;`
4. 如果示例涉及多个函数，函数原型或定义必须在调用前出现

**禁止出现代码片段。** 读者应能将代码直接保存为 `.c` 文件编译运行。

### 标准编译命令

```bash
gcc -std=c99 -Wall -Wextra -g -o example example.c
./example
```

- `-std=c99`：本书统一使用 C99 标准
- `-Wall -Wextra`：开启主要警告，培养良好习惯
- `-g`：包含调试信息，方便 gdb 调试

### 辅助工具命令

```bash
# 涉及动态内存（malloc/calloc/realloc/free）的课程必须提示使用
valgrind --leak-check=full ./example

# 调试时使用
gdb ./example

# 查看预处理结果（预处理器相关课程）
gcc -E example.c
```

### 代码风格

- **缩进**：4 空格（不用 Tab）
- **命名**：变量和函数用 `snake_case`，宏和常量用 `UPPER_SNAKE_CASE`
- **大括号**：K&R 风格（左大括号不换行）
- **注释语言**：注释用中文，解释"为什么"而非"做了什么"
- **每行宽度**：不超过 80 字符

## 二、隐性知识清单

> 本书读者画像：了解 C 基础语法，但对指针高级用法和数据结构实现缺乏实战经验。
> 以下是教材常默认"你已经知道"但目标读者可能不清楚的知识点。**首次涉及时必须主动讲清楚。**

### 内存模型

- 程序内存分为代码段、数据段（全局/静态）、栈、堆四个区域，各有不同生命周期 → module-00 首次提及时用图讲清
- 栈上变量出作用域自动释放，不需要也不能 `free` → module-01 指针入门时讲
- 堆上内存必须手动 `free`，否则泄漏 → module-06 动态内存时系统讲
- 局部变量未初始化时值是不确定的（不是 0）→ module-00 变量声明时提醒
- `malloc` 返回的内存**未初始化**（`calloc` 才初始化为 0）→ module-06 首次使用时强调

### 指针本质

- 指针变量本身也占内存（32 位系统 4 字节，64 位系统 8 字节）→ module-01 内存模型时讲
- `&` 取地址和 `*` 解引用是互逆操作 → module-01 首次出现时用图展示
- `free` 后指针变量仍存在且值不变，但指向的内存已无效（野指针）→ module-06 必须强调并演示
- `NULL` 是一个宏，通常定义为 `((void *)0)`，解引用 NULL 会段错误 → module-01

### 数组与指针关系

- 数组名在大多数表达式中退化（decay）为指向首元素的指针 → module-03 核心知识，必须用图详解
- 退化的两个例外：`sizeof(arr)` 返回整个数组大小、`&arr` 返回指向整个数组的指针 → module-03 必须显式列出
- `char *s = "hello"` 字符串字面量在只读区，不可修改；`char s[] = "hello"` 在栈上，可修改 → module-04 字符串基础时必讲
- 数组作为函数参数时退化为指针，函数内 `sizeof` 得到的是指针大小而非数组大小 → module-03

### 函数调用

- C 语言函数参数**永远是值传递**（传指针也是——传的是地址的副本）→ module-02 函数机制时反复强调
- 函数返回后，其栈上局部变量的地址失效，返回局部变量的指针是未定义行为 → module-02 易错点

### 结构体

- 结构体赋值是浅拷贝（成员逐字节复制），如果成员含指针，两个结构体共享同一块堆内存 → module-05
- `.` 用于结构体变量，`->` 用于结构体指针，`p->member` 等价于 `(*p).member` → module-05 首次使用时对比演示
- 结构体内存对齐导致 `sizeof(struct)` 可能大于各成员之和 → module-05 进阶课讲

### 工具相关

- `gcc -Wall` 不是全部警告，`-Wextra` 覆盖更多 → module-00 首次编译时说明
- `valgrind` 只能检测堆内存错误，栈上越界它检测不到 → module-06 引入 valgrind 时说明
- `gdb` 基础用法：`break`、`run`、`print`、`next`、`step`、`backtrace` → module-01 首次需要调试时简介

### 预处理器

- `#include` 是纯文本替换，把头文件内容原样插入当前位置 → module-09 首次讲时说清楚
- 宏也是纯文本替换，没有类型检查 → module-09 宏与函数对比时强调
- 头文件重复包含会导致重定义错误，所以需要 include guard → module-09

## 三、图表偏好

| 图表类型 | 使用场景 | 推荐方式 | 理由 |
|---------|---------|---------|------|
| 内存布局图 | 栈/堆/全局区、变量存储位置 | ASCII | 需要精确标注地址和字节偏移 |
| 指针关系图 | 指针指向关系、多级指针 | ASCII | 箭头 + 地址框的组合 Mermaid 难以精确控制 |
| 数据结构图 | 链表节点、树节点、数组元素 | ASCII | 需要展示节点内部结构和指针连线 |
| 内存对齐图 | 结构体成员在内存中的排列 | ASCII | 需要按字节精确对齐展示 |
| 编译流程图 | 预处理→编译→汇编→链接 | Mermaid `graph LR` | 线性流程，Mermaid 清晰 |
| 函数调用流程 | 递归展开、回调触发顺序 | Mermaid `graph TD` | 层次关系，Mermaid 适合 |
| 概念关系图 | 概念间依赖/包含关系 | Mermaid `graph TD` | 关系图 Mermaid 擅长 |
| 栈帧变化图 | 函数调用时栈帧的推入和弹出 | ASCII | 需要展示详细的内存地址和变量布局 |

### ASCII 图约定

- 宽度不超过 **80 字符**
- 内存地址用 `0x` 前缀或简化为相对偏移
- 指针用 `-->` 箭头表示指向
- 未初始化/无效内存用 `???` 标记
- 已释放内存用 `[freed]` 标记
- NULL 指针用 `---> NULL` 表示

示例：

```
变量      地址        值           指向
+-----+ 0x1000  +---------+
| ptr |-------->| value=5 |   ptr 指向堆上分配的内存
+-----+         +---------+
                 0x2000
```

## 四、⭐ 深入触发条件

以下内容出现时**必须**使用 `⭐ 深入：` 标记，表示可跳过不影响后续学习：

- **glibc 实现细节**：如 `malloc`/`free` 的 ptmalloc2 内部机制（arena、bin、chunk 结构）
- **汇编层面行为**：如函数调用时栈帧的 `push`/`pop`/`call`/`ret` 指令序列
- **编译器特定行为**：如 GCC 的结构体对齐策略细节、不同优化级别的影响
- **内核系统调用**：如 `brk`/`sbrk`/`mmap` 与用户态 `malloc` 的关系
- **C 标准精确引用**：如 C99/C11 对未定义行为（UB）和实现定义行为的具体条款
- **不同平台差异**：如 32 位 vs 64 位的指针大小、数据类型大小差异的详细讨论
- **历史演变**：如 K&R C vs ANSI C vs C99 的语法差异

以下内容**不标 ⭐**，作为正文讲解：

- 指针的内存模型（这是本书核心）
- `malloc`/`free` 的使用方法和常见错误
- 内存对齐的基本概念和 `sizeof` 的使用
- 函数调用时参数如何传递（概念层面）
- 链表/树等数据结构的实现原理

## 五、[BEGINNER] 补充场景

### 不需要 [BEGINNER] 补充的内容

目标读者已有 C 基础，以下概念**无需** [BEGINNER] 补充：

- 变量声明、基本数据类型（`int`、`char`、`float`、`double`）
- 控制流（`if`/`else`、`for`、`while`、`switch`）
- 基本函数定义和调用
- 基本运算符（算术、比较、逻辑）
- `printf`/`scanf` 的基本使用
- 数组的基本声明和使用

### 可能需要 [BEGINNER] 补充的内容

- **进制表示**：十六进制 `0x` 表示法，在讲内存地址时首次出现可能需要简要解释 → module-01
- **位运算**：`&`、`|`、`^`、`<<`、`>>` 的含义，在讲位段和位操作时 → module-05
- **命令行基本操作**：`gcc` 编译命令的各个参数含义 → module-00 首次编译时
- **Makefile 基础**：如果课程中涉及多文件编译 → module-09（简要提及即可）
- **`void *` 泛型指针**：为什么 `malloc` 返回 `void *` 以及为什么可以赋值给任何指针类型 → module-06
